Tipos de hosting

suele haber varios tipos de hosting estándar apropiados para las necesidades promedio, cada uno esta hecho para adaptarse alas distintas necesidades.

  • Hosting compartidos

    ES el mas común. En un solo servidor se mantienen un muchas de paginas

    estas suelen compartir lso recursos del servidor como ser el CPU y la RAM.

    en la practica se suelen distribuirlos según la configuración del servidor pero teniendo en cuenta la elasticidad para adaptarse a ala demanda

    suelen incluir un panel de control para ajustes del sitio y entregan el mailing de tu pagina

    es gestionado por el proveedor: actualizaciones de software, seguridad y mantenimiento

    el rendimiento se puede ver afectado por la carga del servidor

  • VPS

    virtual private servers

    Son los más comunes

    Por medio de la tecnología de virtualization se pueden crear servidores virtuales para cada web

    La web queda contenida en un servidor virtual y aislada de las demás

    cada vps tiene asignado recursos fijos en cuanto a CPU, RAM. Almacenamiento, garantizando requerimientos mínimos

    el rendimiento NO DEBERÍA verser afectado por la carga del servidor

  • Dedicado

    es un servidor completo real a completa disposición del cliente

    ofrece mucha mejor estabilidad y rendimiento

    el cliente tiene control total del servidor, SO CPU RAM ALMACENAMIENTO SOFTWARE packages actualizaciones

    configuraciones especificas a medida

    requiere mayor conocimiento técnico o tercerizar la administración del servidor

    suelen ser los mas costosos

  • CLOUD hosting

    son granjas de servidores en red con alta capacidad de ejecución y almacenamiento

    otorgan las mayores posibilidades de escalamiento dinámico

    normalmente están distribuidos por el mundo

    tienen escalabilidad que se ajusta a al demanda

    el rendimiento no se ve afectado

    para las empresas masivas que requieren altísimas capacidad de ejecución y rendimiento estable

  • Hosting especializado

    los servidores se adaptan a las necesidades de un servidor we definido con sus características / necesidades propias

    ejemplo WordPress

Tipos de planes

  • compartido: para poca demanda
  • VPS: para considerable a alto
  • Dedicado: para mayor demanda pero poca escalabilidad, ej. trafico alto y estable
  • CLOUD hosting: para alta maxima demanda

escalabilidad

    si se requiere control y personalización se recomienda dedicado o VPS(para un trafico menor)

Estimación de Recursos para Hosting

1. Estimación del Tráfico

Parámetro Descripción Cálculo Ejemplo
Visitas Diarias Número de visitas al día 5,000
Páginas por Visita Número de páginas cargadas por visita 3
Tamaño de Página Tamaño promedio de una página 1 MB
Tráfico Diario (MB) Total de datos transferidos al día Visitas Diarias * Páginas por Visita * Tamaño de Página 5,000 * 3 * 1 MB = 15,000 MB = 15 GB
Ancho de Banda Mensual (GB) Total de datos transferidos al mes (Tráfico Diario (MB) * 30) / 1024 (15,000 MB * 30) / 1024 ≈ 439 GB

2. Estimación de Recursos

Recurso Descripción Estimación Inicial Ejemplo
CPU Procesadores necesarios para manejar la carga 2-4 núcleos 2 núcleos
RAM Memoria para ejecutar el sitio y manejar tráfico 2-4 GB 4 GB
Almacenamiento Espacio para archivos estáticos y base de datos Espacio total necesario + crecimiento 2 GB (archivos) + 1 GB (base de datos) = 3 GB

3. Ajustes y Monitoreo

Realiza pruebas de carga y monitorea el uso de recursos para ajustar tus estimaciones según el rendimiento observado. Considera opciones de escalabilidad para adaptarte al crecimiento del tráfico.

Cómo Determinar el Tráfico Diario de tu Página Web

Puedes elegir alguna de las siguientes opciones según se acomode a tu necesidad

1. Google Analytics

  • Configura Google Analytics en tu sitio web para obtener datos detallados sobre usuarios, sesiones, y páginas vistas.
  • Revisa métricas clave como el número de usuarios únicos, páginas vistas y la tasa de rebote para evaluar el tráfico diario.

2. Archivos de Registro del Servidor

  • Accede a los archivos de registro (logs) de tu servidor web, que contienen información detallada sobre cada solicitud HTTP.
  • Analiza los logs con herramientas como AWStats o Webalizer para conocer el número de visitas, IPs de visitantes, y páginas más visitadas.

3. Herramientas de Monitorización de Rendimiento

  • Usa herramientas como New Relic, Datadog, o Pingdom para monitorear el rendimiento de tu sitio y obtener datos sobre el tráfico.
  • Estas herramientas te permiten ver el número de solicitudes, tiempos de carga y detectar picos de tráfico.

4. Plugins de CMS

  • Si usas un CMS como WordPress, puedes instalar plugins como Jetpack para obtener estadísticas de tráfico directamente en el panel de control.
  • Estos plugins ofrecen información simplificada sobre visitas diarias, páginas populares, y origen del tráfico.

5. Panel de Control del Hosting

  • Muchos proveedores de hosting ofrecen estadísticas de tráfico en su panel de control (por ejemplo, cPanel).
  • Puedes ver información sobre el ancho de banda utilizado, número de visitas, y otras métricas básicas directamente desde el panel.

6. Herramientas de SEO

  • Utiliza herramientas de SEO como SEMrush, Ahrefs, o Moz para obtener estimaciones del tráfico basado en el rendimiento en motores de búsqueda.
  • Estas herramientas también ofrecen información sobre palabras clave, backlinks y ranking, lo que puede influir en el tráfico.

7. Estimación Manual

  • Si conoces el ancho de banda utilizado y el tamaño promedio de tus páginas, puedes hacer una estimación aproximada del número de visitas diarias.
  • Fórmula básica: Número de Visitas = Ancho de Banda Total / Tamaño Promedio de la Página.
